# Hands-On Labs

Practical exercises to reinforce concepts from the curriculum modules.

## How to Use Labs

Each lab includes:
- **Objective**: What you'll learn
- **Prerequisites**: Required knowledge before starting
- **Setup Instructions**: Tools and environment needed
- **Step-by-Step Exercises**: Guided tasks
- **Expected Outcomes**: What success looks like
- **Evaluation Criteria**: How to assess your work
- **Extensions**: Additional challenges for deeper learning

## Lab Categories

### Software Testing Labs (`/software`)
Web application testing, database testing, and system testing scenarios.

### Automation Labs (`/automation`)
Test automation exercises building frameworks and CI/CD integration.

**Available Labs:**
- [API Testing Lab](automation/api-testing-lab.md) - 3-4 hours, Intermediate
  - Test REST APIs using Postman and Python
  - Practice GET, POST, PUT, DELETE requests
  - Write automated API tests with pytest
  - Handle authentication and error scenarios

### Hardware Testing Labs (`/hardware`)
Hardware validation exercises (paper-based, no physical equipment required).

### Systems Testing Labs (`/systems`)
Integration testing for complex systems spanning hardware, firmware, software, and cloud.

## Setup Requirements

### Common Tools

**All Labs:**
- Text editor or IDE (VS Code recommended)
- Git for version control
- Web browser (Chrome recommended)

**Software/Automation Labs:**
- Python 3.7+ or Node.js 14+
- Package manager (pip or npm)
- Postman (free download)

**Hardware Labs:**
- Paper and pencil (for planning exercises)
- Spreadsheet software (Excel, Google Sheets)
- Calculator

## Tips for Success

1. **Read the entire lab first** before starting
2. **Set up your environment** completely before beginning exercises
3. **Take notes** as you work through the lab
4. **Try extensions** for deeper learning
5. **Share your work** in the community for feedback
6. **Don't skip prerequisites** - they're there for a reason

## Getting Help

If you get stuck:
1. Review the relevant module lesson
2. Check the Prerequisites section
3. Read error messages carefully
4. Search for similar issues online
5. Ask in community discussions
