- Code examples are tested and working

- Represents real-world scenarios

## Examples vs Templates

**Templates** (in `/templates`):
- Starting points for new documents
- Include instructions and placeholders
- Generic and broadly applicable
- Empty or minimal content

**Examples** (in `/examples`):
- Complete, filled-out documents
- Show what good looks like
- Specific scenarios and data
- Fully populated content

**Use Templates When:**
Creating new documents for your project

**Use Examples When:**
Learning format, structure, and best practices
